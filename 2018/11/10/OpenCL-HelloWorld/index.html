OpenCL（一） -- HelloWorld

每周技术分享 OpenCL 编程 API 2018/11/10

摘 要

本文主要介绍了OpenCL的基本概念、运行原理、编程API以及适用场景。OpenCL是一个为异构平台编写程序的框架，旨在充分利用GPU强大的并行计算能力以及与CPU的协同工作，更高效的利用硬件高效的完毕大规模的计算。对于N次循环的代码，如for(i=0;i<N;i++),在GPU上，最好的情况下仅仅须要运行一遍，而CPU则需要运行N遍。OpenMPI提供了9个常用的编程API，并且有一套编程定式，系统在运行过程中自行建立CPU与GPU间的通信并分配GPU资源。因此，在实际应用中开发人员仅需编写自己的业务逻辑即可。本文最后例举了两个利用OpenCL提高系统运行效率的实例，以及利用OpenCL查看本地的运行环境的实例。

0x01 OpenCL Basics

OpenCL（Open Computing Language，开放计算语言）是一个为异构平台编写程序的框架，此异构平台可由CPU，GPU，DSP，FPGA或其他类型的处理器与硬件加速器所组成。OpenCL由一门用于编写kernels（在OpenCL设备上运行的函数）的语言（基于C99）和一组用于定义并控制平台的API组成。OpenCL提供了基于任务分割和数据分割的并行计算机制。

OpenCL类似于另外两个开放的工业标准OpenGL和OpenAL，这两个标准分别用于三维图形和计算机音频方面。OpenCL扩展了GPU图形生成之外的能力。OpenCL由非盈利性技术组织Khronos Group掌管。

0x02 OpenCL Operation Model

理论层次：

1、Platform model

1. Basic structure : 连接着一些设备的主机

2. Host:主程序运行的计算单位，比如计算机系统的CPU

3. Device:通过OpenCL库访问的计算单元，比如CPU内的其他核心，GPU，FPGA

4. Further subdivision:设备进步成计算机单元，计算机单元进一步演化成处理元素

2、Execution model

1. Sx,Sy : 单个工作组中的工作项数

2. sx,sy : 工作组中工作项的索引

3. wx,wy : NDRange中工作组的指数

4. Fx,Fy : 全局指数抵消

基本区别:  host执行主程序，Device执行设备内核

设备上的层次结构: NDRange（N维值网格）➡️工作组➡️工作项目

主机定义上下文:设备（仅来自同一平台）
             内核（在设备上执行OpenCL函数）
             程序对象（内核源代码和编译形式的内核）
             内存对象

主机管理队列: 内核执行
            内存对象上的操作
            同步

3、Memory model

Global memory ：all work-items in all work-groups（传参的变量都应当声明__global ）

Constant memory ：在内核执行时，保持constant， 可由host读写

Local memory ：specific work-group ,shared by all work-items in a work-group

Private memory ：only a single work-item

弱一致性模式
  1. 全局和本地存储工作组内的一致性：仅在工作组内的同步点
  2. 全局存储工作组间的一致性：仅在主机级别的同步点

4、Programming model

一个OpenCL的程序实际上就是一系列OpenCL的内核，被这些内核调用的辅函数，

和这些内核被持续调用的过程,支持data parallel 和task parallel的编程模式

Data Parallel
  1. 数据和NDRange间可能的映射：
     · 严格的一对一映射：对一个工作项中的每一个数据元素
     · 更灵活的映射也是有可能的
  2. 支持的设备类别：GPUs

  从可以并发更新的数据元素集合考虑，比如对一系列数做平方处理

Task Parallel
  1. 仅执行一个内核实例（相当于只有一个工作项的NDRange）
  2. 并行通过：
     · 设备上的SIMD（Single instruction Multiple Data）单元（使用OpenCL矢量数据模型）
     · 队列中的多个任务是异步执行的
  3. 支持的设备类别：多核心CPUs，多CPU系统


*其中乱序模式是考虑到自动负载均衡时才会使用*

举个例子，假设你正在写一个简单的跟踪一个班上每个学生成绩的应用。这个应用包含两个主要的任务：1. 为每个学生计算最终成绩，

假定最终成绩是每个学生所有成绩的平均值。 2. 得出班级的平均成绩

这两个任务本身是不可以并行处理的，因为他们两之间是相互依赖的。但尽管1，2之间不能做并行化处理，但跟踪计算每个学生的期末成绩可以。

每个学生的期末成绩如果用常规方式，则用foreach（）依此遍历

但如果用并行的思路，则需要建立隔离的task()；


主机代码中的主要编程步骤：

   1. 确定异构系统的组件
   2. 查询每个组件的特定属性，以在运行时动态调整程序执行
   3. 编译和配置OpenCL内核，内核代码的编程语言OpenCL C
   4. 创建和初始化内存对象（缓冲区，图像）
   5. 使用适合每个内核的最佳设备以正确的顺序执行内核
   6. 收集结果

应用层次：
   • OpenCL Platform API 

   • OpenCL Runtime API

   • OpenCL C (programming language)基于C99，所以C，C++都可以进行编程

0x03 OpenCL Host API

  1. Query platforms : selection

  2. Query devices of the platform : selection

  3. Create context for the devices context 实际上就是OpenCL内核运行的环境

  4. Create queue (for context and device) 用来提交工作给设备，控制一台设备上内核的执行顺序和操作内存对象

  5. Create program object (for context) ← from C string 一个OpenCL的程序对象就是一个代表其OpenCL程序的数据类型

  6. Create memory objects (within context) memory object 是一个全局内存区域的句柄，可以创建用来存储数据，OpenCL里常用buffer和image

  7. Kernel execution:Set kernel arguments，Put kernel into queue : Execution

  8. Copy memory objects with results from device to host(invoke via queue)

  9. Clean up

Query platforms

  • num entries： 数组平台中cl_platform_id类型的预分配元素数
  • platforms：返回有关平台的信息（对于每个平台，数组平台中的一个元素）
  • num_platforms:返回平台数

Query Devices

  • platform : 已经选择的平台
  • device type : 设备类别 (比如 CL_DEVICE_TYPE_CPU, CL_DEVICE_TYPE_GPU)
  • num entries : 数组设备中cl_device_id类型的预分配元素数
  • devices : 返回有关设备的信息（对于每个设备，数组设备中的一个元素）
  • num devices : 返回设备数量

Create Context

  • properties : 位字段，用于定义上下文的所需属性
  • num devices : 应为其创建上下文的设备数
  • devices : 包含要为其创建上下文的设备的数组

  • errcode ret : 返回错误代码（理想情况下等于CL_SUCCESS）

    Create Queue

  • context : 应在其中创建队列的上下文

  • device : 要为其创建队列的设备
  • properties : 用于定义队列所需属性的位字段
  • errcode_ret : 返回错误代码（理想情况下等于CL_SUCCESS）

Create Program Object

  • count : 包含源代码的char缓冲区数量（请参阅字符串）
  • context : 应在其中创建程序对象的上下文
  • strings : 包含指向包含源代码的char缓冲区的指针的数组
  • length : 指定每个char缓冲区长度的数组（以字节为单位）
  • errcode_ret : 返回错误代码（理想情况下等于CL_SUCCESS）

Compile Program

  • num_devices : 程序编译的设备数量

  • device_list : 需要编译程序的设备的数组（这些必须属于与程序对象相同的上下文！）

  • options : 带有编译器选项的Char字符串

Create Kernel

  • program : 包含已编译内核代码的程序对象

  • kernel_name : 内核函数的名称（在程序对象的源代码中）

  • errcode_ret : 返回错误代码（理想情况下等于CL_SUCCESS）

Create Memory Objects

  • context : 应在其中创建缓冲区的上下文

  • flags : 用于定义缓冲区属性和创建时执行的复制操作的位字段

  • size : 缓冲区长度（以字节为单位）

  • host_ptr : 指向主机内存中内存区域的指针，该内存区域用作复制操作的源或直接用于缓冲区

  • errcode_ret : 返回错误代码（理想情况下等于CL_SUCCESS）

Set Kernel Arguments

  • kernel : 设置参数的内核

  • arg_index : 参数的索引（从0开始，用于内核函数的第一个参数）

  • arg_size : 参数值的长度（以字节为单位）

  • arg_value : 指向参数值的指针

Kernel Execution

  • command_queue : 应该用于执行的队列

  • kernel : 要执行的内核

  • work_dim : 数组维数（涉及以下三个参数）

  • global_work_offset: Fx (Fy , Fz )

  • global_work_size : Gx (Gy , Gz ) (所有工作组中每个维度的工作项总数)

  • local_work_size : Sx (Sy , Sz ) (比例Gx /Sx , Gy /Sy , Gz/Sz 需要取整)

Transfer Data from Device to Host

  • command_queue : 应该用于执行的队列

  • buffer : 用作复制操作源的缓冲区对象

  • blocking_read : 如果为true，则该函数仅在复制操作完成后返回（因此，如果它在“按顺序模式”下运行，则也会返回队列中的所有前面的命令）

  • offset : 缓冲区中的读取偏移量（以字节为单位）

  • cb : 要复制的字节数

  • ptr : 指向主机内存中目标区域的指针（需要在之前分配足够的大小）

0x04 应用场景

只要有并行需求，且任务互相独立不依赖，就可以考虑用OpenCL进行并行编程，实现将普通的算法代码放到gpu上运行。

可以跨多平台，mac，windows，Android；

多厂商：NVIDIA，Intel，Apple等等

例1

假设你正在写一个简单的跟踪一个班上每个学生成绩的应用。这个应用包含两个主要的任务：1. 为每个学生计算最终成绩，假定最终成绩是每个学生所有成绩的平均值。 2. 得出班级的平均成绩
这两个任务本身是不可以并行处理的，因为他们两之间是相互依赖的。

尽管1，2之间不能做并行化处理，但跟踪计算每个学生的期末成绩可以，每个学生的期末成绩如果用常规方式，则用foreach（）依此遍历；
但如果用并行的思路，则需要建立隔离的task()；

例2

创建一个4*4的二维空间，按一定规则给每个元素赋值。

实现思路:
选择OpenCL平台并创建一个上下文 clGetPlatformIDs();
clCreateContextFromType();

选择设备并创建命令队列 clCreateCommandQueue();

创建和构建程序对象 clCreateProgramWithSource();
clBuildProgram();

创建内核和内存对象 clCreateKernel(),clCreateBuffer();

设置内核数据并执行内核 clEnqueueNDRangeKernel();

读取执行结果并释放OpenCL资源 clEnqueueReadBuffer(),clReleaseXXX();

实现代码：

// main.cpp
// OpenCL
//
// Created by zhouxin on 2018/11/4.
// Copyright © 2018 zhouxin. All rights reserved.
//

#include <stdio.h>

#include

#include <OpenCL/OpenCL.h>

int main() {
unsigned int i,j;//iterator variables for loops
cl_int err;
cl_platform_id platforms[32];//an array to hold the Ids of all the platforms
cl_uint num_platforms;//number of platforms in this machine
char vendor[1024];//hold platform vendor
cl_device_id devices[16];//hold number of devices for each platform
cl_uint num_devices;
char deviceName[1024];
cl_uint numberofCores;
cl_long amountofMemory;
cl_uint clockFreq;//the clock frequency of a device
cl_ulong maxAlocatableMem;
cl_ulong localMem;
cl_bool available;
char buildinkernels[1024];

clGetPlatformIDs(32, platforms, &num_platforms);
printf("\nNumber of platforms:\t%u\n\n",num_platforms);
for (i=0; i<num_platforms; i++) {
    printf("Platform:\t\t%u\n\n",i);
    err=clGetPlatformInfo (platforms[i], CL_PLATFORM_VENDOR, sizeof(vendor), vendor, NULL);
    printf("\tPlatform Vendor:\t%s (err==%d,%d)\n", vendor, err, CL_SUCCESS);
    err=clGetDeviceIDs (platforms[i], CL_DEVICE_TYPE_ALL, sizeof(devices), devices, &num_devices);
    printf("\tNumber of devices:\t%u  (err==%d,%d) \n\n", num_devices, err, CL_SUCCESS);
    for(j = 0; j < num_devices; j++){
        clGetDeviceInfo(devices[j], CL_DEVICE_NAME, sizeof(deviceName), deviceName, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_VENDOR, sizeof(vendor), vendor, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_MAX_COMPUTE_UNITS, sizeof(numberofCores), &numberofCores, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_GLOBAL_MEM_SIZE, sizeof(amountofMemory), &amountofMemory, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_MAX_CLOCK_FREQUENCY, sizeof(clockFreq), &clockFreq, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_MAX_MEM_ALLOC_SIZE, sizeof(maxAlocatableMem), &maxAlocatableMem, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_LOCAL_MEM_SIZE, sizeof(localMem), &localMem, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_AVAILABLE, sizeof(available), &available, NULL);
        clGetDeviceInfo(devices[j], CL_DEVICE_BUILT_IN_KERNELS, sizeof(buildinkernels), &buildinkernels, NULL);
        printf("\tDevice: %u\n", j);
        printf("\t\tName:\t\t\t\t%s\n", deviceName);
        printf("\t\tVendor:\t\t\t\t%s\n", vendor);
        printf("\t\tAvailable:\t\t\t%s\n", available ? "Yes" : "No");
        printf("\t\tCompute Units:\t\t\t%u\n", numberofCores);
        printf("\t\tClock Frequency:\t\t%u mHz\n", clockFreq);
        printf("\t\tGlobal Memory:\t\t\t%0.00f mb\n", (double)amountofMemory/1048576);
        printf("\t\tMax Allocateable Memory:\t%0.00f mb\n", (double)maxAlocatableMem/1048576);
        printf("\t\tLocal Memory:\t\t\t%u kb\n", (unsigned int)localMem);
        printf("\t\tBuild In Kernels:\t\t%s\n",buildinkernels);
        clGetDeviceInfo(devices[j], CL_DEVICE_OPENCL_C_VERSION, sizeof(buildinkernels), &buildinkernels, NULL);
        printf("\t\tOpenCL Version:\t\t\t%s\n",buildinkernels);
        clGetDeviceInfo(devices[j], CL_DEVICE_PROFILE, sizeof(buildinkernels), &buildinkernels, NULL);
        printf("\t\tDevice profile:\t\t\t%s\n",buildinkernels);
        clGetDeviceInfo(devices[j], CL_DEVICE_VERSION, sizeof(buildinkernels), &buildinkernels, NULL);
        printf("\t\tDevice version:\t\t\t%s\n",buildinkernels);
        clGetDeviceInfo(devices[j], CL_DRIVER_VERSION , sizeof(buildinkernels), &buildinkernels, NULL);
        printf("\t\tDriver version:\t\t\t%s\n",buildinkernels);
        printf("\n");

   }
}
system("PAUSE");
return 0;

}
