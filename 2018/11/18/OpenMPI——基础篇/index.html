OpenMPI——基础篇

每周技术分享 OpenMPI 进程间通信 编程API 2018/11/18

摘 要

本文主要介绍了OpenMPI的概念、架构、编译运行方式、编程定式和基础的、常用的编程API及其用法。最后使用一个简单的Demo来实操这些API，并加深对OpenMPI编程和用途的理解。本文是OpenMPI的基础篇，主要介绍一些基础操作。

0x01 初识OpenMPI

OpenMPI是消息传递接口(Message Passing Interface,MPI)的一种全新的、开源的实现。它可以提供包括单机中进程与进程间、网络中主机与主机间的高性能消息传递。这种高性能主要体现在三个方面:

  1. 高带宽利用率
  2. 低丢包率
  3. 低误码率

OpenMPI可以完美支持并行的、多线程的应用，在消息传递过程中可以检查数据的完整性，能够为应用提供最大化的可用带宽，也可以动态地解决通信过程中设备出现故障的问题。

总之，OpenMPI适用于需要使用多进程处理，且各进程间需要大量通信的环境中。

0x02 OpenMPI的架构及编程定式

OpenMPI的架构可以分为三层(图1)：

  1. MCA
  2. 组件框架(Component frameworks)
  3. 模块(Modules)


图2.1 OpenMPI的架构
  • MCA： 是整个OpenMPI的骨干，主要负责与上层应用交互，根据应用的规定，开启N个进程(其中，N由应用指定)，并将执行的结果返回给应用。
  • 组件框架：由MCA指定，一个组件框架可以视为对应一个进程，实现一项具体的功能或逻辑。根据任务量的不同，组件框架会自动调用(对用户透明)不同数量的模块来完成这一项任务。
  • 模块：真正实现功能的部分，自己实现的功能可以供其他模块调用，每个模块都是相互独立的，属于同一个组件框架的模块实现各自的功能后，由组件框架进行整合，视为完成对应的任务，上交给MCA。

0x03 OpenMPI的HelloWorld!

在编写我们的第一个OpenMPI程序之前，我们先要来认识两个重要的函数：MPI_Comm_size()和MPI_Comm_rank()。

MPI_Comm_size()用于获取总进程数量，其用法如下：

int world_size；

MPI_Comm_size(MPI_COMM_WORLD, &world_size);

运行上述代码之后，变量world_size存储的就是总进程数，若我们敲入“-n 4”的运行指令，运行结果就是world_size = 4。

MPI_Comm_rank()用于获取当前进程号，进程号从0开始，到我们指定的进程数-1。比如4个进程，那他们的进程号就是0、1、2、3。这个函数的用法如下：

int wrank；

MPI_Comm_rank(MPI_COMM_WORLD, &wrank);

考虑到进程是并发运行的，所以对于每一个进程而言，变量wrank为他们的进程号。

总结一下，MPI_Comm_size()是用来获取总进程数的，而MPI_Comm_rank()是获取当前进程号的，这两个函数在后面的章节中很常用。

那么接下来就让我们就开始写HelloWorld吧。

//HelloWorld.c
#include "mpi.h"
#include <stdio.h>
int main(int argc, char**argv) {
    MPI_Init(&argc, &argv); //openMPI的初始化函数
    int world_size, wrank;
    MPI_Comm_size(MPI_COMM_WORLD, &world_size); //获取总进程数
    MPI_Comm_rank(MPI_COMM_WORLD, &wrank); //获取当前进程标识
    printf("Hello world. I'm the process %d, we are %d process in total\n", wrank,world_size);
    MPI_Finalize(); //openMPI的结束函数
    return 0;
}

编程框架在0x02章节已经说明过了，我们主要看看HelloWorld的内容。首先定义了两个变量world_size和wrank，在通过两个函数MPI_Comm_size()和MPI_Comm_rank()给这两个变量赋值，分别为总进程数和当前进程号，最后打印出HelloWorld、当前进程号以及总进程数。考虑到OpenMPI是多进程并发执行，所以当我们指定多进程后，一定会打印出多个HelloWorld。

让我们来编译运行HelloWorld.c源文件，在命令行敲入如下命令：

mpicc -o HelloWorld HelloWorld.c

mpirun -n 4 HelloWorld

使用4个进程运行HelloWorld，其结果如图3所示。



图3.1 HelloWorld

可以看到，当我们使用4个进程运行时，同一段代码被运行了4次，每次运行时打印的常量“HelloWorld”都一样，每次的总进程数也都一样(无论哪个进程在运行，总进程数总是4个)，但是每次运行的当前进程号是不同的，由此可见，每个进程都运行了一次该代码，而变量wrank就对应每一次运行时的进程号。同时可以注意到，进程的并发执行是无序的。

0x04 进程间通信基础

OpenMPI是消息传递接口的一种实现，所以通信是其核心功能。本章节将介绍进程间通信的基础操作，之所以强调基础，是因为还有很多高阶操作，以及不同的通信模式，在一篇文章中写不完……因此，本文先做基础介绍，在接下来的更新中再做高阶的通信介绍。（本章引用的图片均引自OpenMPI学习笔记）

1、点对点通信

点对点通信用于将一组数据从一个进程发送到另一个进程。

//点对点通信函数API
int MPI_Ssend(
    const void *buf,        //要发送的内容，即发送缓冲
    int count,              //发送的数据数量
    MPI_Datatype datatype,  //发送的数据类型    
    int dest,                 //目标进程号，要发给哪个进程
    int tag,                //消息标签，与接收方必须对应相同
    MPI_Comm comm           //一般写MPI_COMM_WORLD
);

int MPI_Recv(
    void *buf,              //要接收的内容，即接收缓冲
    int count,              //接收的数据数量
    MPI_Datatype datatype,  //接收的数据类型
    int source,             //源进程号，接收的是哪个进程的数据
    int tag,                //消息标签，与发送方必须对应相同
    MPI_Comm comm,          //一般写MPI_COMM_WORLD
    MPI_Status *status      //一般写MPI_STATUS_IGNORE
);

其中数据类型与C语言的数据类型可以对应(图4)



图4.1 OpenMPI的数据类型

让我们看一个例子：(为了节省篇幅，以下的所有例子都只给出OpenMPI中的内容，有些部分也可能出现伪代码)

//Example for Transmit
MPI_Init(&argc, &argv);
int wrank;
MPI_Comm_rank(MPI_COMM_WORLD, &wrank); //获取当前进程号
int witness[] = { 0, 0 };  //接收缓冲
if (wrank == 2) {
    int modifier[] = { 1, 1 };  //发送缓冲
    MPI_Ssend(modifier, 2, MPI_INT, 3, 28, MPI_COMM_WORLD); //发送
}
else if (wrank == 3)
    MPI_Recv(witness, 2, MPI_INT, 2, 28, MPI_COMM_WORLD, MPI_STATUS_IGNORE); //接收
printf("Rang %d, witness {%d,%d}.\n", wrank, witness[0], witness[1]); //输出进程号和接收缓冲
MPI_Finalize();

上述代码中，使用if语句块，指定2号进程发送数据，3号进程接收数据，而0、1号两个进程不做操作。在实际编程中，可以根据需要，使用if语句块或switch语句块，分配给不同进程不同的任务，若某一代码段没有指定进程，则该代码段所有进程都需要运行(如所有进程都初始化了接收缓冲)。

运行结果如下：

由于所有进程都初始化了接收缓冲，而只有3号进程接收了数据，所以0、1、2号进程的接收缓冲都是{0,0}，而3号进程的接收缓冲是接收到的{1,1}。

2、全局进程同步

当某个进程遇到全局同步语句后，会等待所有其他进程都运行到该语句，再继续运行其他语句，以达到同步的目的。



图4.2 全局进程同步
//全局同步函数API
MPI_Barrier(MPI_COMM_WORLD);

下面我们来看一个例子：

//Example for Barrier
MPI_Init(&argc, &argv);
int wrank;
MPI_Comm_rank(MPI_COMM_WORLD, &wrank);  //获取当前进程号
if (wrank == 0) {  //进程0写文件
    FILE* f = fopen("outin", "w");
    long int seconds = time(NULL);
    fprintf(f, "%ld", seconds);
    fclose(f);
}
MPI_Barrier(MPI_COMM_WORLD);  //全局同步点
long int witness = 0;
FILE* f = fopen("outin", "r");  //所有进程读文件
fscanf(f, "%d", &witness);
printf("Rang %d, witness %ld.\n", wrank,     witness);
fclose(f);
MPI_Finalize();

运行结果：

在0进程写文件后，进行同步，以保证所有进程等待0号进程写完文件后再读文件，从结果中可见，所有进程读到的结果都相同。如果不进行同步，结果是下面这样的：

进程2和进程3在进程0写文件之前就读到了文件，所以读到的值不相同。

3、广播

广播用于将一个进程的数据发送给包括自己在内的全部进程。



图4.3 广播
//广播API
int MPI_Bcast(
    void *buffer,          //要广播的数据，即发送缓冲
    int count,             //广播的数据数量
    MPI_Datatype datatype, //广播的数据类型
    int root,              //从哪个进程广播
    MPI_Comm comm          //一般写MPI_COMM_WORLD
);

例子：

//Example for Broadcast
MPI_Init(&argc, &argv);
int wrank;
MPI_Comm_rank(MPI_COMM_WORLD, &wrank);  //获取当前进程号
int witness = wrank;  //发送缓冲，要发送的数据为进程号，所有进程都需要这个缓冲区用于收发数据
MPI_Bcast(&witness, 1, MPI_INT, 2, MPI_COMM_WORLD);  //使用2号进程进行广播，注意到发送缓冲和接收缓冲是一致的
printf("Rang %d, witness %d.\n", wrank, witness);  //输出进程号及接收缓冲
MPI_Finalize();

运行结果：

2号进程广播的内容是它的进程号，即数字2。因此，所有进程均收到广播数据2。

4、平均分配

平均分配用于将一个进程的缓冲区中的数据平均发送给包括自己在内的所有进程。



图4.4 平均分配
//平均分配API
int MPI_Scatter(
    const void *sendbuf,     //要分配的数据，即发送缓冲  
    int sendcount,           //分配给每一个进程的数据数量
    MPI_Datatype sendtype,   //分配的数据类型
    void *recvbuf,           //接收缓冲
    int recvcount,           //每一个进程接收到的数据数量
    MPI_Datatype recvtype,   //接收的数据类型
    int root,                //进行分配的进程，哪个进程要发送数据
    MPI_Comm comm            //一般写MPI_COMM_WORLD
);

例子：

//Example for Scatter
MPI_Init(NULL, NULL);
int wrank;
MPI_Comm_rank(MPI_COMM_WORLD, &wrank);  //获取当前进程号
int array[] = { wrank, 2 * wrank, 3 * wrank, 4 * wrank };  //发送缓冲
int witness = 0;  //接收缓冲
MPI_Scatter(array, 1, MPI_INT, &witness, 1, MPI_INT, 2, MPI_COMM_WORLD);  //进行分配
printf("Rang %d, witness %d.\n", wrank, witness);  //输出进程号及接收缓冲
MPI_Finalize();

运行结果：

对于0号进程而言，发送缓冲是{0，0，0，0};对于1号进程而言，发送缓冲是{1，2，3，4}……以此类推。由于指定2号进程发送数据，因此发送的数据是2号进程的{2，4，6，8}，此时进行平均分配，得到如上结果。

5、平均收集

平均收集是平均分配的逆向操作，用于将所有进程的数据汇总到一个进程。



图4.5 平均收集
//平均收集API
int MPI_Gather(
    const void *sendbuf,    //要收集的数据，即每个进程的发送缓冲
    int sendcount,          //每个进程要发送的数据量
    MPI_Datatype sendtype,  //发送的数据类型
    void *recvbuf,          //某一个进程的接收缓冲
    int recvcount,          //从每个进程接收的数据量
    MPI_Datatype recvtype,  //接收的数据类型
    int root,               //哪个进程进行接收
    MPI_Comm comm           //一般写MPI_COMM_WORLD
);

例子：

//Example for Gather
MPI_Init(NULL, NULL);
int wrank;
MPI_Comm_rank(MPI_COMM_WORLD, &wrank);  //获取当前进程号
int witness[] = { 0, 0, 0, 0 };   //接收缓冲，每一个进程都创建并初始化
MPI_Gather(&wrank, 1, MPI_INT, witness, 1, MPI_INT, 2, MPI_COMM_WORLD);  //进行平均收集
printf("Rang %d, witness", wrank);  //输出进程号及接收缓冲
for (int i = 0; i < 4; ++i)
    printf(" %d", witness[i]);
printf(".\n");
MPI_Finalize();

运行结果：

可以看到，我们把进程号作为要发送的数据，2号进程进行收集，所以0、1、3号进程的接收缓冲没有变动，都是初始化时的{0，0，0，0}，而2号进程的接收缓冲是收集到的{0，1，2，3}。

6、普遍收集

普遍收集用于将所有进程发送的数据收集给包括自己在内的所有进程，相当于平均收集和广播的结合。



图4.6 普遍收集
//普遍收集API
int MPI_Allgather(
    const void *sendbuf,    //要收集的数据，即发送缓冲
    int sendcount,          //每一个进程要发送的数据数量
    MPI_Datatype sendtype,  //发送的数据类型
    void *recvbuf,          //接收的数据，即接收缓冲
    int recvcount,          //从每一个进程接收的数据数量
    MPI_Datatype recvtype,  //接收的数据类型
    MPI_Comm comm           //一般写MPI_COMM_WORLD
);

例子：

//Example for Allgather
MPI_Init(NULL, NULL);
int wrank;
MPI_Comm_rank(MPI_COMM_WORLD, &wrank);  //获取当前进程号
int witness[] = { 0, 0, 0, 0 };  //接收缓冲
MPI_Allgather(&wrank, 1, MPI_INT, witness, 1, MPI_INT, MPI_COMM_WORLD);  //进行普遍收集
printf("Rang %d, witness", wrank);  //输出进程号及接收缓冲
for (int i = 0; i < 4; ++i)
    printf(" %d", witness[i]);
printf(".\n");
MPI_Finalize();

运行结果：

可以看出，相比上一节的平均收集，普遍收集会将结果收集到包括自己在内的所有进程。

7、选择性分配

平均分配要求分配的数量必须相同，而对于选择性分配，分配的数量可以不同。因此，平均分配可以视为选择性分配的一种特殊情况。



图4.7 选择性分配
//选择性分配API
int MPI_Scatterv( 
    const void *sendbuf,     //发送缓冲
    const int sendcounts[],  //要发送给各个进程的数据数量，注意是一个数组
    const int displs[],      //要发送给各个进程的数据所在缓冲区的起始位置，也是一个数组
    MPI_Datatype sendtype,   //发送的数据类型
    void *recvbuf,           //接收缓冲
    int recvcount,           //接收的数据数量
    MPI_Datatype recvtype,   //接收的数据类型
    int root,                //哪一个进程进行分配
    MPI_Comm comm            //一般写MPI_COMM_WORLD
);

看代码之前，先来了解两个重要的数组：sendcounts[] 和 displs[]。

  • sendcounts[]用于表示从0号进程开始，要分配给各个进程的数据数量。如图4.7所示，对应的sendcounts[] = {1,2,1,2}，表示0号进程发送一个，1号进程发送两个，2号进程发送一个，3号进程发送两个。

  • displs[]用于表示从0号进程开始，要发送给各个进程的数据所在缓冲区的起始位置。如图4.7所示，对应的displs[] = {0,1,3,4}，表示分配给0号进程的数据起始在0位置，分配给1号进程的数据起始在1位置，分配给2号进程的数据起始在3位置，分配给3号进程的数据起始在4位置。

利用sendcounts[]和displs[]两个数组，相当于标定了发送给各进程的数据所在缓冲区的起始位置及长度，由此可以确定需要发送的数据内容。

理解了这两个数组的作用，就可以来看例子了(获取当前进程号的语句已经省略)：

//Example for Scatterv
MPI_Init(NULL, NULL);
int recvbuf[] = { wrank, wrank };  //接收缓冲，初始化为两个当前进程号
int recvcount = (wrank % 2) + 1;   //接收数据的数量，偶数号进程接收1个，奇数号进程接收两个
int sendcounts[] = { 1, 2, 1, 2 }, //发送的数据数量
int displs[] = { 0, 1, 3, 4 };     //发送的数据所在缓冲区的起始位置
int total = 6;                     //发送的数据总数
if (wrank == 2) sendbuf[] = {1000 to 1005};  //伪代码，手动设置要发送的数据内容
MPI_Scatterv(sendbuf, sendcounts, displs, MPI_INT,     recvbuf, recvcount, MPI_INT, 2, MPI_COMM_WORLD);  //进行选择性分配
printf("Rang %d, recvbuf %d %d.\n", wrank, recvbuf[0],     recvbuf[1]); //输出进程号及接收缓冲
MPI_Finalize();

运行结果：

从结果可以看出，1、3号进程接收了两个数据；而0、2号进程仅接收一个数据，从而第二个位置的数据没有变动，是初始化时的当前进程号。每个进程接收到的数据的顺序也和代码中一致。

8、选择性收集

选择性收集是选择性分配的逆向操作，用于将不同数量的数据收集到某个进程。平均收集是选择性收集的一种特殊情况。



图4.8 选择性收集
//选择性收集API
int MPI_Gatherv( 
    const void *sendbuf,     //发送缓冲
    int sendcount,           //每一个进程要发送的数据数量
    MPI_Datatype sendtype,   //发送的数据类型
    void *recvbuf,           //接收缓冲
    const int recvcounts[],  //接收的数量，类比上一节的sendcounts[]
    const int displs[],      //接收的起始位置，类比上一节的displs[]
    MPI_Datatype recvtype,   //接收的数据类型
    int root,                //哪一个进程接收
    MPI_Comm comm            //一般写MPI_COMM_WORLD
);

recvcounts[]和displs[]两个数组可以参考上一节，这里不再赘述，直接看例子：

MPI_Finalize();

运行结果：

0号进程的发送缓冲为{0，0}，要发送的数据量是一个，因此仅发送一个数字0；1号进程的发送缓冲为{1，1}，要发送的数据量是两个，因此发送了两个数字1，1……以此类推。2号进程接收到的数据根据recvcounts[]和displs[]两个数组决定，由此得出上述结果。

9、行列互换

行列互换用于将每一个进程的数据平均分发给包括自己在内的所有进程，宏观上看就相当于行与列相互交换。



图4.9 行列互换

    const void *sendbuf,     //发送缓冲
    int sendcount,           //发送的数量
    MPI_Datatype sendtype,   //发送的类型
    void *recvbuf,           //接收缓冲

    MPI_Datatype recvtype,   //接收类型
    MPI_Comm comm            //一般写MPI_COMM_WORLD
);

例子：

//Example for Alltoall
MPI_Init(NULL, NULL);
int power = 10 * wrank + 10;   //生成与当前进程号有关的数据
int sendbuf[] = {power,power+1,power+2,power+3};  //发送缓冲
int recvbuf[] = { 0, 0, 0, 0 };   //接收缓冲
MPI_Alltoall(sendbuf, 1, MPI_INT, recvbuf, 1, MPI_INT, MPI_COMM_WORLD);  //进行行列互换
//打印每个进程的发送缓冲和接收缓冲
MPI_Finalize();

运行结果：

观察进程号、发送缓冲及接收缓冲后可以发现，如果将进程号按照0、1、2、3的顺序排列，则相当于做了一次矩阵的行列互换。其本质是0号进程做了一次平均分配，然后1号进程再做平均分配……以此类推。

0x05 简单Demo

第四章介绍了基础的、常用的进程间通信函数，本章通过一个简单的综合性例子，来看看如何使用这些函数。

现在要求：由一个进程生成一组数列，要求计算这组数列的第1-5项平均值及差，第6-8项的最大值，第9-12项的最小值，以及第13-14项的乘积，将结果保存到一个新的数组中，将该数组告知所有进程。

一个简单的思路，是由0号进程(哪个进程都无所谓)生成一组数列，使用选择性分配，将这个数列分配给包括自己在内的其他进程；由于第1-5项需要求平均值和差两项内容，所以可以由0号进程将1-5项通过点对点的方式发给另一个进程；当所有进程并发计算完成后，将结果通过平均收集的方式收集到0号进程(同样，哪个进程都无所谓)，最终使用广播的方式告知所有进程。

整个过程参考图5.1



图5.1 计算过程

完整代码如下：

//simpleDemo.c
#include <mpi.h> 
#include <stdio.h> 
#include <stdlib.h>
int main(int argc, char **argv) {
    MPI_Init(NULL, NULL);
    int world_size, wrank;
    MPI_Comm_size(MPI_COMM_WORLD, &world_size);
    MPI_Comm_rank(MPI_COMM_WORLD, &wrank);

    /* 1、选择性分配阶段 */
    const int total = 14;               //要发送的总数据量
    int recvbuf[5] = { 0,0,0,0,0 };     //接收缓存
    int recvcount = 5;                  //接收的数据量
    int sendcounts[] = { 5, 3, 4, 2 };  //发送的数据量
    int displs[] = { 0, 5, 8, 12 };     //要发送的数据所在的起始位置
    int sendbuf[total];                 //发送缓存
    if (wrank == 0) {
        /* 生成发送序列 */
        for (int i = 0; i < total; i++) {
            if (i < 5) sendbuf[i] = 100 + i;
            else if (i < 8) sendbuf[i] = 1000 + 2 * i;
            else if (i < 12)sendbuf[i] = 10 + 10 * i;
            else sendbuf[i] = 11;
        }
        /* 显示发送序列 */
        printf("Rang %d, sendbuf ", wrank);
        for (int i = 0; i < total; i++) {
            printf("%d ", sendbuf[i]);
        }
        printf("\n");
        //sendbuf[] = { 100 101 102 103 104 1010 1012 1014 90 100 110 120 11 11 };
    }
    MPI_Scatterv(sendbuf, sendcounts, displs, MPI_INT, recvbuf, recvcount, MPI_INT, 0, MPI_COMM_WORLD);

    /* 显示发送的结果 */
    printf("Rang %d, recvbuf ", wrank);
    for (int i = 0; i < 5; i++) {
        printf("%d ", recvbuf[i]);
    }
    printf("\n");

    MPI_Barrier(MPI_COMM_WORLD);

    /* 2、数据处理阶段 */
    int sendbuff, sum, max, min, diff;
    int receivebuff[5];
    switch (wrank) {
    case 0:
        MPI_Ssend(recvbuf, 5, MPI_INT, 4, 28, MPI_COMM_WORLD);  //将同样的数据发送给4号进程
        sum = 0;
        for (int i = 0; i < 5; i++) sum += recvbuf[i];
        int avg = sum / 5;
        sendbuff = avg;
        break;
    case 1:
        max = recvbuf[0];
        for (int i = 1; i < 3; i++)
            if (max < recvbuf[i]) max = recvbuf[i];
        sendbuff = max;
        break;
    case 2:
        min = recvbuf[0];
        for (int i = 1; i < 4; i++)
            if (min > recvbuf[i]) min = recvbuf[i];
        sendbuff = min;
        break;
    case 3:
        sendbuff = recvbuf[0] * recvbuf[1];
        break;
    case 4:
        MPI_Recv(recvbuf, 5, MPI_INT, 0, 28, MPI_COMM_WORLD, MPI_STATUS_IGNORE);  //接收0号进程的数据
        diff = recvbuf[0];
        for (int i = 1; i < 5; i++)
            diff -= recvbuf[i];
        sendbuff = diff;
        break;
    default:
        break;
    }
    printf("Rang %d, sendbuff %d\n", wrank, sendbuff);

    /* 平均收集阶段 */
    MPI_Gather(&sendbuff, 1, MPI_INT, receivebuff, 1, MPI_INT, 0, MPI_COMM_WORLD);
    if (wrank == 0) {
        printf("Rang %d, recvbuff ", wrank);
        for (int i = 0; i < 5; i++) {
            printf("%d ", receivebuff[i]);
        }
        printf("\n");
    }

    /* 广播阶段 */
    MPI_Bcast(receivebuff, 5, MPI_INT, 0, MPI_COMM_WORLD);
    printf("Rang %d, receivebuff ", wrank);
    for (int i = 0; i < 5; i++) {
        printf("%d ", receivebuff[i]);
    }
    printf("\n");

    MPI_Finalize();
    return 0;
}

输入编译运行指令：

$ mpicc -o simpleDemo simpleDemo.c
$ mpirun -n 5 simpleDemo
